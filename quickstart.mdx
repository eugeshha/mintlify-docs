## Как пользоваться этой докой

Это **личная учебная документация**. Цель — собрать в одном месте всё необходимое для перехода в DevOps: теории, команды, примеры и ссылки. Контент можно дополнять своими заметками и чек-листами.

<Steps>
  <Step title="Запусти превью локально (по желанию)">
    В корне папки `mintlify-docs` выполни:
    ```bash
    npm i -g mint
    mint dev
    ```
    Сайт откроется на `http://localhost:3000`. Удобно для навигации и поиска.
  </Step>
  <Step title="Выбери точку входа">
    - **Новичок в администрировании** — начни с раздела [Основы](/fundamentals/linux) (Linux, Git, скрипты, сети).
    - **Уже умеешь Linux и Git** — можно идти в [Docker](/containers/docker) и [Kubernetes](/containers/kubernetes).
    - **Хочешь обзор всего пути** — открой [Дорожную карту](/roadmap).
  </Step>
  <Step title="Иди по разделам и практикуй">
    В каждом разделе есть теория и практические шаги. После каждой темы пробуй команды и сценарии у себя (VM, облако, домашний кластер).
  </Step>
  <Step title="Дополняй контент">
    Добавляй в страницы свои примеры, ссылки на курсы/статьи и чек-листы — дока станет твоей базой знаний.
  </Step>
</Steps>

## Рекомендуемый порядок обучения

1. **Основы** — Linux, Git, скрипты (Bash/Python), основы сетей.
2. **Контейнеры** — Docker (образы, Dockerfile, docker-compose), затем Kubernetes (pods, deployments, services).
3. **CI/CD** — одна система на выбор (например, GitHub Actions), потом расширять на GitLab CI / Jenkins.
4. **IaC** — Terraform (облачная инфраструктура), Ansible (конфигурация и деплой).
5. **Облако** — базовые сервисы одного провайдера (например, AWS), запуск K8s в облаке.
6. **Мониторинг** — метрики (Prometheus + Grafana), логи, алерты.
7. **Безопасность** — секреты, сканирование образов/кода, основы DevSecOps.
8. **Карьера** — роль DevOps, сертификаты, подготовка к собесам.
