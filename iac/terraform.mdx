## Роль Terraform

- **Что это**: инструмент описания и управления инфраструктурой в виде кода.
- **Для чего**: создавать, изменять и удалять облачные ресурсы по декларативным описаниям.

## Ключевые концепции

- **Конфигурации**: файлы с описанием ресурсов и их свойств.
- **State**: состояние уже созданных ресурсов, с которым синхронизируется Terraform.
- **Providers**: плагины для работы с конкретными платформами (облака, Kubernetes и др.).

## Практическое применение

- **Повторяемое создание инфраструктуры**: одинаковые окружения для dev, stage, prod.
- **Контроль изменений**: планирование (`plan`) и применение (`apply`) с явным diff.
- **Интеграция с CI/CD**: автоматическое управление ресурсами при выкатывании приложения.
