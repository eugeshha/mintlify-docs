## Цели CI/CD

- **Continuous Integration (CI)**: автоматически проверять и объединять изменения в основной кодовой базе.
- **Continuous Delivery/Deployment (CD)**: автоматически выкатывать проверенный код на целевые среды.

## Основные компоненты

- **Пайплайн**: последовательность шагов (build → test → deploy).
- **Агенты выполнения**: среды, в которых запускаются шаги.
- **Артефакты**: результаты сборки (образы, пакеты, бандлы).

## Практическое применение

- **Снижение ручной работы**: выкат по кнопке или по событию в Git.
- **Повторяемость**: одинаковые шаги для всех веток и окружений.
- **Контроль качества**: автоматические тесты и проверки перед деплоем.
